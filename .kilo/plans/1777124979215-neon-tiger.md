# Project Stack Overview

## Framework & Runtime
- **Next.js 16.2.2** (React 19.2.4, React DOM 19.2.4)
- **TypeScript** ^5
- **ESLint** ^9 with `eslint-config-next`

## Styling & UI
- **Tailwind CSS** ^4 (via `@tailwindcss/postcss`)
- **Shadcn/UI** components (using Radix UI primitives)
  - `@radix-ui/react-dialog` ^1.1.15
  - `@radix-ui/react-select` ^2.2.6
  - Additional UI components (button, card, input, label, etc.) sourced from local `src/components/ui/`
- **Tailwind utilities**
  - `class-variance-authority` ^0.7.1
  - `clsx` ^2.1.1
  - `tailwind-merge` ^3.5.0
- **Icons**: `lucide-react` ^1.7.0

## Backend & Data
- **Supabase** client `@supabase/supabase-js` ^2.101.1
- Supabase migrations managed via SQL files in `supabase/migrations/`

## Build & Dev Tools
- Next.js CLI (`next dev`, `next build`, `next start`)
- No custom Babel/Webpack configuration (uses Next.js defaults)
- ESLint for linting (`npm run lint`)

## Configuration Files
- `next.config.ts`: Custom redirects (legacy Japanese nuru massage paths)
- `tsconfig.json`: Standard TypeScript configuration (not inspected but present)
- `package.json`: Defined scripts and dependencies above
- `.env.local`: Environment variables (Supabase credentials, etc.)
- `.env.local.example`: Example environment file

## Project Structure Highlights
- `src/app/`: App Router structure with route groups `(public)` and `(admin)`
- `src/lib/`: Supabase client (`client.ts`, `server.ts`) and utility functions (`utils.ts`)
- `src/components/`: Reusable UI components (adsense, UI primitives)
- `supabase/migrations/`: Database schema evolution scripts

## Summary
This is a modern Next.js 16 + React 19 application using TypeScript, Tailwind CSS with Shadcn/UI primitives, Supabase as the backend, and a curated set of utility libraries for styling and iconography. The project follows Next.js App Router conventions with public and admin sections separated via route groups.
